# Дизайн: автообновление + русский интерфейс (Stom client)

**Дата:** 2026-06-12 · **Статус:** согласовано · **Релиз:** v0.1.5

## Проблема
1. **Баг (срочно):** на машине пользователя закэширован старый engine-pack без
   `multiprocessing.freeze_support()` → локальная сегментация падает с traceback
   про `freeze_support` (см. `aa.png`). Клиент не отслеживает версию пакета и
   никогда его не обновляет, поэтому фикс из v0.1.4 не доезжает.
2. **Нет автообновления** ни клиента, ни движка.
3. **Английский интерфейс**, местами непонятный; ошибки показываются «портянкой».

## Решения (согласовано с пользователем)
- Автообновлять **и движок (engine-pack), и приложение (.exe)**.
- Механизм: **спрашивать пользователя** (не молча).
- Русский UI: **полный перевод + перекомпоновка**, без переключателя языков.

## Архитектура
«Тонкий вид + тестируемое ядро» (как в текущем проекте). Вся логика проверки
версий — в модулях без Qt, покрыта юнит-тестами; `main_window` только показывает
диалоги и запускает фоновые воркеры.

### 1. Обновление движка (engine-pack) — чинит текущую ошибку
- `engine_pack.provision()` после успешной установки пишет маркер
  `engine_dir()/installed.json` = `{"version", "sha256"}`.
- `engine_pack.installed_version() -> str | None` читает маркер; `None`, если
  маркера нет (старая/сломанная установка).
- `engine_pack.engine_update_available(manifest) -> bool`: `True`, если версия
  манифеста ≠ установленной **или** маркера нет, но exe присутствует (legacy).
- `provision(..., clean=False)`: при `clean=True` очищает `root` перед
  распаковкой, чтобы не оставались файлы старого пакета.
- Эффект: текущая машина (legacy без маркера) сама получит предложение обновиться
  до актуального engine-pack.

### 2. Обновление приложения (.exe)
- Новый модуль `stomclient/updates.py` (без Qt):
  - `current_version() -> str` через `importlib.metadata.version("stomcore")`.
  - `parse_version(tag) -> tuple[int,...]` (срезает `v`, парсит semver).
  - `fetch_latest_release(opener=urlopen) -> dict` — GitHub API
    `repos/Almaz2001AA/Stom/releases/latest` → `{"version": tag, "url": <asset>}`.
  - `client_update_available(current, latest) -> bool` (semver-сравнение).
  - Сеть недоступна/ошибка → исключение ловится выше, проверка молча пропускается.
- UI при старте (фон): если новее → диалог «Доступна новая версия vX.
  Скачать и установить?». По согласию: скачать `StomClientSetup.exe` с прогрессом
  во временную папку, запустить его и закрыть приложение.
- В Inno-спеку добавить `CloseApplications=yes` (и `RestartApplications=no`),
  чтобы установщик мог заменить файлы работающего приложения.

### 3. Русский интерфейс + перекомпоновка
- Все видимые строки → русский, централизованы в `stomclient/ui/strings.py`
  (словари `BTN`, `MSG`, `STATUS`, `TITLE`).
- Левая панель — группы с заголовками-разделителями:
  - **Исследование:** «Открыть DICOM…»
  - **Сегментация:** галка «Локально (на этом ПК)»; «Сегментировать» /
    «Загрузить и сегментировать»; «Установить движок…» / «Обновить движок…»
  - **Инструменты:** «Плоскость» (combo: переименовать значения),
    «Измерение», «Очистить измерения»
  - **Экспорт:** «Сохранить PNG…», «Сохранить маску…»
  - «Настройки…» (сверху), список масок «Маски:» (снизу), статус-строка.
- `State` → русские подписи через `STATUS` (покрыть все значения enum).

### 4. Понятные ошибки
- Диалог ошибки сегментации: короткое сообщение + «Подробности» (раскрывающийся
  текст с traceback), вместо стены текста. Реализация — `QMessageBox` с
  `setDetailedText()`.
- Если ошибка локального движка содержит признаки старого пакета (`freeze_support`)
  → текст «Движок устарел» + предложение «Обновить движок».

## Тестирование
- `engine_pack`: маркер пишется/читается; `engine_update_available` (другая
  версия / нет маркера+exe → True; та же версия → False; ничего не установлено →
  без падений); `clean=True` удаляет старые файлы.
- `updates.py`: `parse_version`, `client_update_available` (новее/старее/равно);
  `fetch_latest_release` с инъектированным opener (без сети).
- UI: `STATUS` покрывает все `State`; окно собирается с русскими подписями
  (обновить существующие UI-тесты, где ассертится английский текст).
- Тонкий вид сохраняется: логика версий — вне Qt.

## Вне объёма (YAGNI)
- Переключатель языков EN/RU (только русский).
- Дельта-обновления / фоновая авто-установка без подтверждения.
- Самозамена .exe без установщика.

## Выпуск
Релиз **v0.1.5** (тег пересоберёт клиент и engine-pack). Начиная с него
обновления приходят сами по схеме выше. v0.1.2/v0.1.3 — сломаны, не использовать.
